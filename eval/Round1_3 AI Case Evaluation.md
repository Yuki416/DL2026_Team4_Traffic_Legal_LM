### 2. 核心爭議（⚠️ 模板複讀，嚴重過擬合）

**LoRA 模型三題輸出比較：**

| | TC-01（左轉） | TC-02（無號誌巷口） | TC-03（閃紅燈） |
|---|---|---|---|
| 爭議 1 | 是否左轉時未禮讓直行車 | 是否**左轉**時未禮讓直行車 ← 情境無左轉 | 是否於閃光紅燈路口未停車再開 |
| 爭議 2 | 駕駛過失與被害人受傷結果是否具有相當因果關係 | 駕駛過失與被害人受傷結果是否具有相當因果關係 | 是否**左轉**時未禮讓直行車 ← 情境無左轉 |
| 爭議 3 | — | — | 駕駛過失與被害人受傷結果是否具有相當因果關係 |

**問題確認：**
- 「是否左轉時未禮讓直行車」在 TC-02（無號誌巷口，根本無左轉）和 TC-03（閃紅燈，根本無左轉）被硬套
- 「相當因果關係」出現在三題，是訓練資料高頻模板的直接複讀
- 模型沒有「讀懂情境再分析」，而是「觸發關鍵詞後套模板」

**對照 LawBot TW 的處理：**
- TC-02 正確識別「無號誌路口路權優先順位（左方車讓右方車）」和「減速慢行義務」，完全情境特異
- TC-03 正確識別「閃光紅燈＝停車再開」的具體義務，而非泛用說法

### 3. 責任類型（❌ 三題完全相同，民事概念缺失）

**LoRA 模型三題輸出：**
```json
["刑事過失傷害責任", "違反道路交通安全注意義務"]
```
三題**一字不差**，與情境完全無關。

**缺失的責任面向：**
- TC-01：應有民事損害賠償 + 過失相抵比例（70–80% vs 20–30%）
- TC-02：應有民事賠償 + 自首減刑選項
- TC-03：應有民事賠償 + 過失相抵（60–70%）+ 強制險扣除

訓練資料以 NTDM 刑事判決為主，民事賠償案例幾乎缺席，導致模型的「責任類型」欄位完全被刑事覆蓋。

---

### 4. 適用法條（⚠️ 精度不足，情境法條缺失）

| 情境 | 應有法條 | LoRA 輸出 | 問題 |
|---|---|---|---|
| TC-01 | 道交規則 **102-1-7**（轉彎讓直行）、94-3 | 102條第1項（無款） | 款項不精確；漏 94-3 |
| TC-02 | 道交規則 **102-1-2**（左讓右）、**93-1-2**（減速慢行）、刑法 284 | 102條第1項（無款）、刑法 284 | 款項不精確；漏 93-1-2（最關鍵的義務） |
| TC-03 | 道交規則 102-1-1、號誌規則 **211** | 刑法 284（TC-03 沒有刑事）、號誌規則 211 | 幻覺：TC-03 未提刑事，模型自行添加刑法 284 |

**補充說明 TC-03 的幻覺問題：**
LawBot TW 對 TC-03 的回覆刻意**未提刑法 284**（對方輕傷，主要是民事賠償情境）。原始 Llama 也提到了刑法 284，顯示「提到輕傷就套刑法 284」是兩個模型共有的傾向，但 LawBot TW 有能力根據情境克制這個傾向，我們的模型則不然。

## 綜合診斷

### 成功的部分
- **格式遷移完整**：341 筆資料就讓模型完全學會三欄 JSON 結構，原始 Llama 做不到
- **語氣與術語**：輸出的措辭是法律文件語氣，不是散文或口語
- **TC-03 核心爭議**：第一條「是否於閃光紅燈路口未停車再開並禮讓幹道車」是情境正確的

### 失敗的部分
| 問題 | 嚴重程度 |
|---|---|
| 責任類型三題完全相同（純刑事，無民事） | 🔴 嚴重 |
| 核心爭議跨情境複讀模板（左轉套在無左轉情境） | 🔴 嚴重 |
| 法條只到條未到款，情境關鍵法條（93-1-2）缺失 | 🟡 中等 |
| TC-03 幻覺添加不適用的刑法 284 | 🟡 中等 |

## 根本原因分析

| 觀察問題 | 根本原因 |
|---|---|
| 模板複讀 | 訓練資料 139 筆（41%）重複描述 → 模型學到「核心爭議 = 那幾句固定套語」 |
| 責任類型只有刑事 | 資料集以刑事過失傷害判決為主，幾乎無民事賠償案例 |
| 法條不精確到款 | 32% 法條為 GPT-4o 推斷，粒度不夠細 |
| loss 0.145 是過擬合訊號 | 341 筆 × 3 epochs，資料少跑多輪 → 記憶不是泛化 |
| 過擬合無從察覺 | 無 Validation Set，無法早停 |

**能力遺忘警訊：**
原始 Llama 的 TC-02 散文輸出中，有「未注意右側狀況」「減速慢行」等情境正確的推理。微調後模型反而把它覆蓋成「左轉未讓直行」的模板。格式訓練帶來了情境推理能力的退化，這是 Round 1 最需要注意的副作用。

## 對 Round 2 的建議

**最高優先（資料集品質）：**
1. **打破 139 筆重複描述**，改成多樣化問法
2. **補充民事賠償案例**（這是目前最嚴重的缺口）
3. **法條精確到款**，從判決書原文抽取而非 GPT-4o 推斷

**訓練策略：**
4. **切出 Validation Set**（至少 10%），讓過擬合可被偵測
5. **epochs 降到 1–2**，或使用 early stopping 根據 val loss 停止
6. 三月份 1,443 筆（資料量 4 倍）本身就能緩解過擬合

**評估方法：**
7. 以判決書 Ground Truth 測試集（~145 筆）建立可量化的評估，才能把「模板複讀程度」轉成可追蹤的指標
