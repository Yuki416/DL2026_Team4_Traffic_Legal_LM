## 背景

二月份原始篩選只跑了刑事 pipeline，且未過濾整案被駁回（和解撤告）的判決，導致訓練集有兩個問題：

1. **民法完全缺席（0/341 筆）**：只從刑事資料庫篩選，刑民分離，民法條文根本不會出現
2. **含無實質內容的駁回案件**：`本件公訴不受理` 是和解後撤告的結案方式，這類判決沒有法院事實認定，法條引用也幾乎空白，拿來訓練是雜訊

三月份同時解決了這兩個問題，套用相同邏輯重篩二月份資料，可以得到更乾淨的刑事資料集 + 新的民事資料集。

---

## 刑事 Pipeline（`02_filter.py`）

三層條件 + 整案駁回過濾，**全部同時滿足才保留**：

**第一層：案類**

```python
# JCASE 欄位含這些子字串（子字串比對，壢交簡、審交易等地區變體自動涵蓋）
"交" in doc["JCASE"]
```

**第二層：案由**

```python
TARGET_TITLES = ["過失傷害", "過失致死", "過失致重傷", "過失重傷害"]
# JTITLE 欄位含其中一個
```

> 注意：二月份原版漏掉了「**過失重傷害**」（少了「致」字），重篩時要確認這四個都在。

**第三層：路口路權關鍵字**

```python
PATH_KEYWORDS = ["路口", "轉彎", "直行", "支線道", "幹線道",
                 "閃紅燈", "閃黃燈", "禮讓", "貿然", "未讓"]
# JFULL 判決書全文含其中一個（OR）
```

**額外過濾：整案被駁回**

```python
FULL_DISMISS_PATTERNS = ["本件公訴不受理", "本案公訴不受理"]
CONVICTION_PATTERNS   = ["處有期徒刑", "處拘役", "處罰金"]

# 排除邏輯：若 JFULL 含駁回模式，且完全沒有任何定罪語句 → 排除
# （若有部分定罪 + 部分不受理，代表是刑事案件中有和解部分但仍有判決，保留）
if 含駁回模式 and not 含任何定罪語句:
    排除
```

這個過濾會移除和解後撤告的案件（無法院事實認定、無法條引用）。三月份因此從 1,443 降到 **899 筆**，移掉的都是低品質空殼。

## 民事 Pipeline（`02b_filter_civil.py`）

兩層條件，**不使用 JCASE 篩選**（民事 JCASE 超過 30 種地區代碼，無法靠 JCASE 判斷是否交通案件）：

**第一層：案由**

```python
JTITLE_TRAFFIC_MARKER = "交通"
JTITLE_TYPES = ["損害賠償", "侵權行為"]
# JTITLE 同時含「交通」且含「損害賠償或侵權行為」

# 排除補繳裁判費程序裁定（JCASE 結尾為「補」，含中補、北補、板補等地區變體）
if jcase.endswith("補"):
    排除
```

**第二層：事故內容關鍵字**

```python
CIVIL_KEYWORDS = [
    # 沿用刑事 10 個
    "路口", "轉彎", "直行", "支線道", "幹線道",
    "閃紅燈", "閃黃燈", "禮讓", "貿然", "未讓",
    # 民事判決書慣用詞（新增 6 個）
    "車禍", "碰撞", "撞擊", "交通事故", "闖紅燈", "肇事",
]
# JFULL 含其中一個（OR）
```

> 民事判決書不用「貿然」這種刑事法律套語（命中率只有 22%），主力是「交通事故」（71%）和「碰撞」（61%）。**不能只用刑事的 10 個關鍵字**，否則會漏掉大量純財損賠償案件。

## 執行流程

```
1. 拿出二月份原始資料（JY202602 的 JSON 檔）
2. 跑 02_filter.py（改 DATA_ROOTS 指向 202602）→ traffic_cases_Feb_criminal.json
3. 跑 02b_filter_civil.py（改 DATA_ROOT 指向 202602）→ traffic_cases_Feb_civil.json
4. 以 JID 去重後合併 → traffic_cases_Feb_merged.json
```

執行前只需要把腳本裡的路徑從 `202603` 改成 `202602`，其他邏輯完全相同。

## 注意事項

### 路徑統一

目前兩支腳本路徑格式不一致，跑之前要先確認：

- `02_filter.py` 用相對路徑：`data/raw/202602`
- `02b_filter_civil.py` 用絕對路徑：`/mnt/8tb_hdd/under115b/.../raw/202602`

建議改成同一種格式，確保兩支腳本讀的是同一批原始資料。

### Output Schema 建議

重篩後若進入 GPT-4o 抽取，Prompt 裡要明確要求民事面向，否則民事判決的民法內容還是出不來。最低限度是在 instruction 裡加：

> 「若判決書涉及民事賠償，請在責任類型欄位中包含過失相抵比例與賠償範圍」
