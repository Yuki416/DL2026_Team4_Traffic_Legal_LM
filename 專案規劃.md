# 專案目標

> **打造一個「車禍肇責初步諮詢」的台灣本土法律 AI——民眾用大白話描述車禍經過，模型立刻回傳「這是誰的錯（核心爭議）+ 涉及哪些法律責任（刑/民/行）+ 相關法條」。**

## 任務定義

**輸入白話文災情描述 ➔ 輸出核心爭議點 + 責任類型 + 推薦適用法條**

模型吃進民眾的口語車禍描述，輸出三樣結構化結果：

1. **核心議題**：這場車禍的肇責爭議是什麼（10 個標籤，見下）
2. **責任類型**：涉及哪些法律責任（刑事 / 民事 / 行政）
3. **推薦法條**：相關的道安規則、刑法、民法、處罰條例

### 運行範例

**輸入（民眾大白話）：**

> 「我今天騎車綠燈直行過馬路，對面一台轎車突然要左轉，根本沒打方向燈就直接撞上我。我當場摔倒在地、腳大面積擦挫傷，機車外殼也全碎了。結果對方司機下車還一直大聲罵我，說路口本來就要減速，是他先轉彎轉到一半的，堅持不肯賠錢，請問這真的是我的錯嗎？」

**輸出（模型結構化分析）：**

1. **核心議題**：轉彎車未讓直行車 / 交通事故肇責歸屬
2. **責任類型**：刑事過失傷害 + 民事侵權損害賠償
3. **建議法條**：
   - 《道路交通安全規則》第 102 條（轉彎車應讓直行車先行）
   - 《刑法》第 284 條（過失傷害罪）
   - 《民法》第 184 條（侵權行為損害賠償責任）

# 一、我們要處理什麼樣的議題？

聚焦在 **路口路權與肇責歸屬爭議**——車禍裡最常見、最會起爭執的那一類：

- 轉彎車 vs 直行車（誰該讓誰）
- 支線道 vs 幹線道（沒有號誌的路口）
- 閃紅燈 vs 閃黃燈、紅燈右轉
- 與有過失（雙方都有錯，責任比例怎麼分）

**刻意不處理**的類型：

| 排除類型 | 原因 |
|---|---|
| 酒駕（公共危險） | 爭議是「有沒有醉」，不是「誰的路權」 |
| 肇事逃逸 | 爭議是「有沒有逃」，不是路口責任歸屬 |
| 單純違規無傷亡 | 沒有肇責爭議 |

換句話說，我們處理的是「綠燈直行被左轉車撞，到底是誰的錯？」這種民眾真的會問的問題，而不是所有交通法律問題。

## 議題分類體系（10 標籤）

1. 轉彎車未讓直行車
2. 支線道未讓幹線道
3. 閃紅燈未讓閃黃燈
4. 左方車未讓右方車
5. 紅燈右轉／闖紅燈
6. 號誌違規（其他）
7. 與有過失（過失相抵）
8. 過失傷害成立
9. 過失致死／致重傷
10. 民事損害賠償請求

## 主要法條範圍

- **行政**：道路交通安全規則 §102、道路交通管理處罰條例 §45/§48
- **刑事**：刑法 §276、§284
- **民事**：民法 §184、§191-2、§193、§195、§217

# 二、這個議題為什麼重要？

**因為一般民眾遇到車禍時，資訊極度不對等，又求助無門。**

| 痛點 | 說明 |
|---|---|
| **車禍太普遍** | 幾乎每個有駕照的人一輩子都會碰上一兩次，但絕大多數人完全不懂路權和肇責怎麼判 |
| **當下最焦慮** | 事故現場對方一口咬定是你的錯、堅持不賠，你心慌又沒概念，不知道自己站不站得住腳 |
| **問律師太貴、太慢** | 為了一件擦撞去花錢諮詢律師不划算，但網路資訊雜亂、不一定適用台灣法規 |
| **肇責有規則可循** | 路權歸屬在《道路交通安全規則》有明文，法院判決也有穩定的認定套路 |

最後一點正是本專案可行的關鍵：**肇責判斷有結構化知識可以學**，所以我們能從司法院判決書萃取出「事故描述 → 肇責認定 → 適用法條」的訓練配對。

# 三、我們希望模型如何幫助一般民眾？

讓不懂法律的人，在車禍發生後能 **馬上得到一個有依據的初步判斷**：

| 民眾的處境 | 模型提供的幫助 |
|---|---|
| 「對方說是我的錯，我也不確定」 | 點出**核心爭議**：這是「轉彎車未讓直行車」的典型情況 |
| 「我要負什麼責任？會被告嗎？」 | 說明**責任類型**：可能涉及刑事過失傷害 + 民事損害賠償 |
| 「有沒有法律依據？」 | 列出**適用法條**：道安規則 §102、刑法 §284、民法 §184 |

## 產品定位

**「初步諮詢／自我評估工具」**

- 幫民眾在面對對方、報警、找保險或律師之前，先有個底，知道自己大概站在哪一邊、該注意什麼
- **不是取代律師**，而是降低「連自己有沒有理都不知道」的資訊門檻

> ⚠️ 因為民眾問的「我會怎樣」其實橫跨三種法律後果（會不會被判刑、要不要賠錢、會不會被開罰單），所以**責任類型必須包含刑 / 民 / 行三種**。這也是資料集需要同時涵蓋刑事、民事（、行政）判決的根本原因。
