# 如何結構化抽取判決書

本文件說明 **Step 3（結構化抽取）** 如何從司法院裁判書全文（`JFULL`）中，
用**規則 / 正規表達式**抽取出四個結構化欄位：`事故摘要`、`核心爭議`、`責任類型`、`適用法條`。

> **設計原則：全程規則式、零 LLM。**
> 好處是免費、可解釋、無幻覺；代價是事故摘要的定位偶爾不完美（詳見下方）。

對應腳本：[`scripts/03_extract.py`](scripts/03_extract.py)

## 1. 事故摘要 — 段落定位 + 正規表達式抓取

刑事案件走 `extract_fact_section_criminal()`，**四層策略，由精準到粗略**，
任一層成功就回傳：

| 策略 | 做法 | 這筆案件的結果 |
|---|---|---|
| 1 | 找「附件」後的犯罪事實段（聲請簡易判決書常見） | 無附件 → 失敗 |
| 2 | 找正文「犯罪事實」或「事　實」標題段 | JFULL 有「　　事　實」→ **命中，回傳「一、蘇瑞聲…」段落** |
| 3 | 全文搜 `ACCIDENT_PATTERN`（需要日期錨點） | 策略 2 已成功，不執行 |
| 4 | 全文搜 `ACCIDENT_PATTERN_LOOSE`（只要 駕駛/騎乘…碰撞） | 策略 2 已成功，不執行 |

民事案件走 `extract_fact_section_civil()`，策略類似（詳見後半段民事章節），
第一層找「理由要領」格式、第二層找「原告主張」段。

### 兩個關鍵 pattern

```python
# 精準：要有「於民國113年7月21日…駕駛…碰撞」這樣的日期錨點
# 支援「於114年」（無民國前綴）與「於民國114年」兩種寫法
ACCIDENT_PATTERN = re.compile(
    r"(?:於(?:民國)?\d{2,4}年)\d+月\d+日.{0,50}(?:駕駛|騎乘|行駛)"
    r".{5,200}(?:碰撞|撞擊|擦撞|追撞|肇事|發生車禍|交通事故|受傷|死亡)",
    re.DOTALL
)

# 粗略：不要求日期，只要「駕駛/騎乘…(300字內)…碰撞/受傷」
ACCIDENT_PATTERN_LOOSE = re.compile(
    r"(?:駕駛|騎乘).{5,300}(?:碰撞|撞擊|擦撞|追撞|肇事|車禍|交通事故|受傷)",
    re.DOTALL
)
```

### 策略 2 如何抽出完整段落

`FACT_HEADERS_CRIMINAL` 包含 `"　　事　實\n"`（全形縮排版），
本案 JFULL（正規化 `\r\n`→`\n` 後）在位置 247 命中此標題。

命中後從標題後取 1200 字的 segment，對 `一[、．,，]\s*(.+?)(?:\n二[、．,，]|\Z)` 做 `re.search`：
命中「一、蘇瑞聲…」並抓取到「二、」之前的整段，包含從主被告行為到碰撞、致死的完整事實。
清理空白後長度 548 字，`truncate_at_sentence(max_chars=600)` 不截斷，直接回傳。

**結果：摘要從蘇瑞聲（主被告）開始，語意完整。**

> **過去的限制（已修）**：早期 `FACT_HEADERS_CRIMINAL` 只含「犯罪事實」變體，
> 未含「事　實」，Strategy 2 對本案失敗，最終落到 Strategy 4（ACCIDENT_PATTERN_LOOSE），
> 從謝○霖（共同被告）那句「騎乘」開始。加入「事　實」系列標題後此問題消失。

### 截斷處理

抓到的段落最後再經過 `truncate_at_sentence()`：在 600 字以內找最後一個句尾
（`。` 或 `；`）切斷，避免在句子中間硬截。找不到句尾才硬截到 600 字。
（函數預設值是 400，但腳本中所有呼叫均傳入 `max_chars=600`。）

```python
def truncate_at_sentence(text: str, max_chars: int = 400) -> str:
    if len(text) <= max_chars:
        return text
    window = text[:max_chars]
    last_end = max(window.rfind("。"), window.rfind("；"))
    if last_end > 50:
        return text[:last_end + 1]
    return window
```

## 2. 核心爭議 — 固定清單關鍵字比對（不自由生成）

走 `match_disputes()`，掃全文比對 **12 條固定規則**，
按 `DISPUTE_ORDER` **由具體到抽象**，**最多取 3 個**。

規則有兩種型態：
- **單字串**：該字串出現即命中
- **tuple**：tuple 內所有字串都要出現才命中

```python
DISPUTE_RULES = {
    "支線道未讓幹線道":   ["支線道", "幹線道"],
    "閃紅燈未停讓":      ["閃紅燈"],
    "閃黃燈未減速":      ["閃黃燈"],
    "左方車未讓右方車":  ["左方車未讓", ("左方", "右方")],
    "轉彎車未讓直行車":  ["轉彎車未讓", ("轉彎", "直行"), ("左轉", "未讓"), ("右轉", "未讓")],
    "變換車道未讓直行車": ["變換車道", ("變換", "直行")],   # 新增
    "未注意車前狀況":    ["未注意車前狀況", "未保持安全距離", "車前狀況"],
    "違反號誌":         ["闖紅燈", ("紅燈", "號誌"), "闖越紅燈"],
    "與有過失":         ["與有過失", "過失比例", ("被害人", "過失"), "過失相抵", ("訴外人", "過失")],
    "過失致死致重傷":   ["過失致死", "過失致重傷", "死亡"],
    "過失傷害成立":     ["過失傷害"],
    "民事損害賠償":     ["損害賠償", ("賠償", "民事")],
}
```

### 這筆案件的比對過程

| 順序 | 標籤 | 規則 | 全文是否命中 |
|---|---|---|---|
| 1–4 | 支線道 / 閃紅燈 / 閃黃燈 / 左方車 | — | ✗ |
| **5** | 轉彎車未讓直行車 | `("轉彎","直行")` | ✓「**轉彎**車應讓**直行**車先行」→ 第 1 個 |
| 6 | 變換車道未讓直行車 | `"變換車道"` | ✗ 本案是轉彎，無「變換車道」 |
| **7** | 未注意車前狀況 | `"車前狀況"` | ✓「應注意**車前狀況**」→ 第 2 個 |
| 8 | 違反號誌 | `("紅燈","號誌")` / `闖紅燈` | ✗ 有「紅燈」但無「號誌」 |
| **9** | 與有過失 | `("被害人","過失")` 等 | ✓（謝○霖超速亦有過失）→ 第 3 個 |

湊滿 3 個就 `break`，所以輸出正好是這三個，順序也與 `DISPUTE_ORDER` 一致。

> **關鍵優點：標籤來自固定清單，模型訓練資料裡不會出現幻覺的爭議類型。**
> 代價是無法涵蓋清單外的特殊情境（例如「酒駕」「跨越雙黃線」目前沒有對應規則）。

## 3. 責任類型 — 依來源 + 全文關鍵字

走 `determine_responsibility()`：

```python
def determine_responsibility(record):
    source = record["_source"]
    jfull  = record["JFULL"]
    if source == "criminal":
        if "賠償" in jfull and "損害賠償" in jfull:
            return ["刑事", "民事"]
        return ["刑事"]
    else:
        return ["民事"]
```

- `_source` 由篩選 pipeline 決定（刑事 pipeline → `criminal`，民事 pipeline → `civil`）
- 刑事案件**必有「刑事」**；若全文同時出現「賠償」與「損害賠償」，再加上「民事」
- 純民事案件不看內文，直接回 `["民事"]`

### 這筆案件

`_source` 是 `criminal` → 必有「刑事」。
主文有緩刑條件「依附件調解筆錄…**支付損害賠償**」，
全文同時含「賠償」與「損害賠償」→ 加上「民事」。
結果為 `["刑事", "民事"]`。

## 4. 適用法條 — 三方法抽取 + 去重 + 濾程序法

走 `extract_laws()`，**只抓全文實際出現的文字，不推斷補入**。

### 抽取（三方法）

1. **已知法律名稱 + 阿拉伯數字**（主力）
2. **已知法律名稱 + 中文數字**（補充部分判決書）
3. **fallback regex**（前兩者全空時才啟用，抓罕見法律）

已知法律名稱清單按**長度排序先比對長名稱**，
避免「中華民國刑法」被「刑法」搶先切碎；`中華民國刑法` 會標準化為 `刑法`。

```python
ARTICLE_SUFFIX_ARABIC = r"第(\d+(?:-\d+)?\s*條(?:第\d+項)?(?:第\d+款)?(?:前段|後段|但書)?)"
```

### 去重 `_dedup_laws()`

這筆案件正好示範去重效果。全文出現了**同一條文的不同精細度**：

```
道路交通管理處罰條例第86條          ← 被移除（被更具體版本涵蓋）
道路交通管理處罰條例第86條第1項      ← 被移除（被更具體版本涵蓋）
道路交通管理處罰條例第86條第1項第1款  ← 保留
道路交通管理處罰條例第86條第1項第2款  ← 保留
```

去重邏輯：若存在「**以我為前綴、且下一字是 `前 / 後 / 但 / 第`**」的更具體版本，
就刪掉我自己。

- 泛化的 `第86條`、`第86條第1項` 被吃掉
- 第 1 款、第 2 款是**平行的兩款**（互不為前綴），都保留

### 濾程序法

全文一定會出現「刑事訴訟法第299條」之類的程序條文，
但這些是「怎麼跑法院程序」，不是交通實體法，會被整批過濾：

```python
PROCEDURAL_LAWS = ["刑事訴訟法", "民事訴訟法", "行政訴訟法", "強制執行法"]
```

### 這筆案件最終留下 7 條

| 法條 | 意義 |
|---|---|
| 道路交通管理處罰條例第86條第1項第1款 | 行政加重（無照） |
| 道路交通管理處罰條例第86條第1項第2款 | 行政加重（超速等） |
| 刑法第276條 | 過失致死本罪 |
| 刑法第62條前段 | 自首減刑 |
| 刑法第71條第1項 | 數罪併罰 |
| 刑法第74條第1項第2款 | 緩刑宣告 |
| 刑法第284條前段 | 過失傷害（另一被害人） |

## 1. 事故摘要 — 四層策略

民事案件走 `extract_fact_section_civil()`，共四層策略（與刑事相似，標題清單不同）：

| 策略 | 做法 | 這筆案件的結果 |
|---|---|---|
| 0 | 找「理由要領」格式（小額簡易判決，無「原告主張」段） | 本案無此格式 → 失敗 |
| 1 | 找「原告主張」系列標題（含一審、上訴審變體） | **命中「原告主張：」→ 成功** |
| 2 | 找「事實及理由」段，搜尋 `原告主張` 子字串或 `ACCIDENT_PATTERN` | 策略 1 已成功，不執行 |
| 3 | 全文搜 `ACCIDENT_PATTERN` / `ACCIDENT_PATTERN_LOOSE` | 策略 1 已成功，不執行 |

### 策略 1 的標題清單

```python
FACT_HEADERS_CIVIL = [
    # 一審：帶冒號優先（確保 start 不會停在「：」上）
    "一、原告起訴主張：", "一、原告起訴主張:", "一、原告起訴主張",
    "一、原告主張：",     "一、原告主張:",     "一、原告主張",
    "原告起訴主張：",     "原告起訴主張:",
    "原告主張：",         "原告主張:",
    # 上訴審：原告改稱「被上訴人」或「上訴人」
    "一、被上訴人主張：", "一、被上訴人主張:", "一、被上訴人主張",
    "被上訴人主張：",     "被上訴人主張:",
    "一、上訴人主張：",   "一、上訴人主張:",   "一、上訴人主張",
    "上訴人主張：",       "上訴人主張:",
]
```

### 為何這筆案件用「原告主張：」命中

本案文字是「**二、原告主張：**被告於民國112年…」。開頭是「二、」不是「一、」，
但清單裡的 `原告主張：`（全形冒號）作為**子字串**仍然命中（位於 JFULL 第 456 字）。
命中後從標題後切，往後找 `\n三、`、`\n（一）`、`\n被告` 等作為結束點，抓出整段，
清理後 448 字，語意完整。

**結果：摘要從被告（無照駕駛人）開始，包含事故經過與求償聲明。**

> 民事判決普遍有結構工整的「原告主張：」段，命中率高；
> 刑事有「事　實」標題清單涵蓋，兩者在有對應標題時品質相當。
> 真正品質較差的是：刑事沒有標題的案件（靠 ACCIDENT_PATTERN 定位），
> 以及民事的「理由要領」小額判決（策略 0 靠 pattern 而非標題）。

## 2. 核心爭議 — 命中 3 個規則

這筆案件對全文跑 12 條規則後**命中 3 個**：

| 標籤 | 規則 | 命中？ | 為什麼 |
|---|---|---|---|
| 轉彎車未讓直行車 | `("轉彎","直行")` | ✗ | 全文有「**直行**車」但**沒有「轉彎」**——本案是**變換車道**不是轉彎 |
| **變換車道未讓直行車** | `"變換車道"` | ✓ | 全文有「欲**變換車道**」→ 第 1 個 |
| 未注意車前狀況 | `"車前狀況"` / `"未保持安全距離"` | ✗ | 全文寫「注意安全距離」，不含這些固定字串 |
| 違反號誌 | `"闖紅燈"` / `("紅燈","號誌")` | ✗ | 全文無紅燈相關文字 |
| **與有過失** | `("訴外人","過失")` | ✓ | 全文有「訴外人杜志峯」且多次出現「過失」→ 第 2 個 |
| **民事損害賠償** | `"損害賠償"` | ✓ | 第 3 個，達上限 break |

> **歷史紀錄**：v1 版規則清單沒有「變換車道未讓直行車」，「與有過失」也沒有 `("訴外人","過失")` trigger，
> 這筆案件在當時只能命中「民事損害賠償」1 個。兩條規則補齊後現在命中 3 個。

## 3. 責任類型 — 民事最單純

```python
else:  # source == "civil"
    return ["民事"]
```

`_source` 是 `civil` → **直接回 `["民事"]`，連內文都不看**。這欄在民事案件零誤判。

## 4. 適用法條 — 幽靈法條 bug 已修

抽取結果四條，均正確：

```
道路交通管理處罰條例第21條第1項第1款  ← 正確（無照駕駛）
強制汽車責任保險法第29條第1項第5款   ← 正確（代位求償依據）
道路交通安全規則第98條第1項第6款     ← 正確（變換車道義務）
民法第184條第1項前段               ← 正確（侵權行為一般條款）
```

### 歷史：曾抽到幽靈法條「保險法第29條第1項第5款」

v1 版本曾多抽一條「保險法第29條第1項第5款」。根因：`KNOWN_LAW_NAMES` 同時有
「強制汽車責任保險法」和「保險法」，每個 pattern 各自獨立 `finditer` 全文，
「保險法」的 regex 命中長名稱內部的「保險法第29條…」子字串：

```
強制汽車責任[保險法第29條第1項第5款]   ← 「強制汽車責任保險法」完整命中
              [保險法第29條第1項第5款]   ← 「保險法」也獨立命中 → 幽靈
```

### 修法：span 重疊過濾

`extract_laws()` 現已在收集所有 match 後，執行 **span 包含關係過濾**：
若 match A 的原文位置範圍 `[s_a, e_a]` 完全落在 match B 的 `[s_b, e_b]` 之內
（且兩者 span 不完全相同），則 A 為幽靈法條，丟棄。

```python
def is_ghost(idx):
    _, s_a, e_a = all_matches[idx]
    for j, (_, s_b, e_b) in enumerate(all_matches):
        if idx == j:
            continue
        if s_b <= s_a and e_a <= e_b and (s_b < s_a or e_b > e_a):
            return True
    return False
```

「保險法」match 的 span 完全在「強制汽車責任保險法」match 之內 → 被正確識別為幽靈並移除。

# 民事 vs 刑事 萃取對照

| 欄位 | 刑事 | 民事 |
|---|---|---|
| 事故摘要 | 「犯罪事實」段，標題變體多、易被打斷 | 「原告主張」段，**通常更乾淨完整** |
| 核心爭議 | 過失情境豐富，常滿 3 個 | 多落在「民事損害賠償」，標籤偏薄 |
| 責任類型 | 看內文決定刑事 or 刑事+民事 | **固定 `["民事"]`** |
| 適用法條 | 刑法為主 | 民法+保險法，幽靈法條 bug 已修 |

# 總結：各欄位的可靠度

| 欄位 | 方法 | 可靠度 | 說明 |
|---|---|---|---|
| 責任類型 | 來源旗標 + 關鍵字 | **高** | 邏輯單純，誤判率極低 |
| 適用法條 | 正規表達式抓原文 + 去重 + 濾程序法 | **中高** | 不無中生有；長法名幽靈法條 bug 已修（span 重疊過濾） |
| 核心爭議 | 固定清單關鍵字比對 | **中高** | 無幻覺，但可能漏清單外情境（如酒駕、跨越雙黃線） |
| 事故摘要 | 段落定位 + pattern 抓取 | **中** | 刑事偶爾抓到共同被告；民事較乾淨。需 Step 4 改寫修飾 |

整套流程**規則式、零 LLM**：

- **優點**：免費、可解釋、無幻覺、可重現
- **代價**：事故摘要起點偶爾不完美、核心爭議無法涵蓋清單外情境

這些不完美之處，由後續 **Step 4（白話化改寫）** 在生成訓練配對時進一步修飾。
幽靈法條 bug 已透過 span 重疊過濾修正，不再是待處理問題。
